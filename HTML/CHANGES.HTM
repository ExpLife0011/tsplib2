TSP++ Version 2.24 (June 2000)

 
Changes for this release new    fix    compatibility change

Note: all changes are for 32-bit version only

Changed the CTSPICallAppearance::Drop function to return a zero return code when lineDrop is called multiple times with a pending drop operation in effect.  Previous versions returned LINEERR_OPERATIONUNAVAIL and cause Microsoft Outlook 2000 to report an error.  This is likely a bug in Outlook as it is calling lineDrop multiple times for the same call without waiting for the result, but TSP++ has changed to stop the error box from appearing.

Changed the CServiceProvider::ConvertDialableToCannonical to ignore numbers which are not fully 10-digits. This is for international and internal PBX support.

Corrected a bug in the ATSP sample when lineConfig is used to configure a specific line device.

Added a check to ensure that the highest negotiated version available is TAPI 2.1 to preserve the proper structure length for the pre-built librares.  Added the TAPI_CURRENT_VERSION constant to both samples.

Corrected a bug in calling the CanSupportCall function when processing the LINECALLPARAMS structure.

Corrected a bug when sizing structures for Unicode.
