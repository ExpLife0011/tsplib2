TSP++ Version 2.24 (June 2000)

Introduction

Welcome to TSP++ Version 2.0 - the easiest way to create 1st party Telephony Service Providers for Windows. Please read these release notes carefully as they give additional or updated information from the documentation.

Build/Test Environment

We only support Microsoft Visual C++ 6.0 at this time.  The product should compile under older versions of Visual C++, back to version 4.2, but we do not guarantee or test these compilers. We also strongly recommend that you update your compiler to the Visual Studio service pack 2 or better for this product. The pre-built binary libraries were built with Visual Studio service pack 3 which is available from the Microsoft website.

If you are using any other version of Visual Studio other than 6.0 with service pack 3 installed, you must recompile the libraries to ensure the same versions of the CRT and MFC are being used as much of it is inline implementations and will generally not work across versions in binary form. In addition, if you have any other SDKs which change the headers for the MFC or CRT libraries, you should also recompile the library to ensure that it matches any TSP that you build with it.  This includes the Microsoft Platform SDK CDs or updates through MSDN.

The 16-bit development libraries require the final version of Microsoft Visual C++ for 16-bit Windows (1.52c).  This was shipped with MSDN (Disc 1 -- Development Tools and Microsoft Office Professional for Windows 95 January 1997).   It was also shipped with VC++ 4.2.  If you have neither of these, you can contact Microsoft and attempt to order the product if it is still available.

TAPI 1.x

We recommend that you do the majority of your development and testing in the Windows 95 environment without the TAPI 2.1 update installed.  This will eliminate possible problems with the thunking layer used in TAPI 2.1. 

TAPI 2.x

We recommend that you do the majority of your development and testing in the Windows NT 4.0 checked build environment with NT symbols installed.  This will allow you to trace the interactions between your TAPI applications, the TAPI system, and your TSP.  We also recommend updating to Windows NT Service Pack 4 or better (at this writing, Service Pack 5 has been released and is available) in order to get the most stable version of TAPI.

Provided Samples

The samples come in two different builds -- one for TAPI 1.4 and one for TAPI 2.x. Please make sure to install the appropriate version if you want to play with either sample.

To install the samples:

Windows 3.1 and Windows 95 with TAPI 1.x:

  1. Run the INSTALL.EXE program from the TSPLIB/16BIT/SAMPLES/BIN directory and choose which providers to install or deinstall.
  2. If you installed DSSP (the digital sample), then run the EMULATOR.EXE program from the 16-bit samples directory. Make sure to configure the emulator; for information on how to do this, consult the documentation.

Windows NT, Windows 98 and Windows 95 with TAPI 2.x or Windows 2000 with TAPI 2.0:

  1. CD to the TSPLIB/32BIT/SAMPLES/BIN directory.
  2. Copy the files (ATSP32.TSP, DSSP32.TSP) to the WINNT\SYSTEM32 or WINDOWS\SYSTEM directory.
  3. Run the Telephony Control panel applet and "Add a Provider".
  4. If you installed DSSP32 (the digital sample), then run the EMULATOR.EXE program from the 32-bit samples directory. Make sure to configure the emulator; for information on how to do this, consult the documentation.
